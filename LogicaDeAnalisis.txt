MANUAL TÉCNICO DE ANÁLISIS E INTERPRETACIÓN DEL DAT (DIFFERENTIAL APTITUDE TESTS)
INTRODUCCIÓN
Este documento detalla la lógica completa para procesar, corregir e interpretar la Batería de Aptitudes Diferenciales. El objetivo es transformar la cadena de datos crudos (Raw Data) proveniente del Frontend en un Informe de Competencias detallado.

PARTE 1: ARQUITECTURA DE DATOS Y LÓGICA DE CORRECCIÓN (EL MOTOR)
Esta sección define cómo "piensa" el sistema al recibir los datos. Establece las reglas matemáticas para convertir las respuestas del usuario en un Puntaje Bruto (PB).

1.1. Desglose del Input (La Cadena de Entrada)
El sistema recibe un string con este formato: {PI: Tiempo - 1;val, 2;val...} {PII: Tiempo - 1;val...} {Tiempo - 1;A, 2;B...}

Lógica de Mapeo de Bloques: Para que el análisis sea correcto, el sistema debe identificar qué bloque corresponde a qué test. Según tu estructura HTML/PDF, el orden estricto es:
Orden,Bloque ID,Test Correspondiente,Tipo de Dato,Cant. Ítems,Lógica de Corrección

1.2. Algoritmos de Corrección por Módulo
A. Lógica para Test 1: Rapidez y Exactitud Perceptiva (PSA)
Principio: Este test mide velocidad de procesamiento. No hay "razonamiento".
Fórmula de Puntaje: PB = (Aciertos Parte I + Aciertos Parte II).

Manejo de Errores: En este test, los errores suelen ser por "desliz" visual. No restan puntos en la versión web moderna, pero indican baja atención al detalle.

Validación:

Input Usuario vs Target.

Si Input == Target -> +1 Punto.

Nota: El Target es el valor subrayado en el cuadernillo. El Input es lo que seleccionó el usuario.

B. Lógica para Tests de Potencia (Verbal, Numérico, Abstracto)Principio: Miden capacidad intelectual superior.Fórmula de Puntaje: PB = Sumatoria de Aciertos.Corrección de Azar (Opcional Avanzado): Si deseas aplicar rigor psicométrico estricto para evitar que el usuario "adivine", la fórmula es: $PB = A - \frac{E}{n-1}$ (Donde A=Aciertos, E=Errores, n=Número de opciones).Recomendación para tu sistema: Usa Solo Aciertos. Es el estándar en evaluaciones corporativas modernas para no frustrar al usuario.

C. Lógica para Test de Ortografía (Lógica Inversa)
Principio: Reconocimiento de errores.

Regla: La opción correcta es la palabra MAL ESCRITA.

Si el usuario marca la palabra que está bien escrita -> 0 Puntos.

Si el usuario marca la palabra que tiene el error -> 1 Punto.

PARTE 2: LÓGICA DE NORMALIZACIÓN (DEL PUNTAJE AL SIGNIFICADO)
El analista no puede reportar "Sacó 30 puntos". Eso no significa nada sin contexto. Esta parte define cómo transformar ese número en un Nivel de Competencia.

2.1. Definición de Escalas (La Regla de Tres Psicométrica)
Debemos convertir el Puntaje Bruto (PB) en un Percentil (PC).
Definición: El percentil indica qué porcentaje de la población está por debajo del usuario.

Ejemplo: Percentil 80 significa que el usuario supera al 80% de los candidatos.

Una vez que el sistema determina el nivel, debe asignar una "Etiqueta Semántica" para el informe:

PARTE 3: LÓGICA DE ANÁLISIS INTEGRAL (PERFILES COMBINADOS)
Esta es la sección más importante para el "Analista". Aquí cruzamos los datos. No analizamos los tests por separado, sino cómo interactúan entre sí.

3.1. El Factor "Aptitud Académica" (VR + NR)
Lógica: Sumar el puntaje de Razonamiento Verbal (VR) + Razonamiento Numérico (NR).

Significado: Es el mejor predictor de éxito en roles administrativos, ejecutivos y de aprendizaje general.

Reglas de Inferencia:

Si ambos son ALTOS: Candidato de alto potencial (High Potential). Aprende rápido cualquier tarea. Ideal para gerencia.

Si VR Alto y NR Bajo: Perfil humanista, legal, ventas, comunicación. Puede fallar en presupuestos o lógica financiera.

Si VR Bajo y NR Alto: Perfil técnico puro, contador, analista de datos. Puede tener problemas para comunicar sus hallazgos o liderar equipos.

3.2. El Factor "Inteligencia Fluida" (AR + NR)
Lógica: Combinación de Abstracto (AR) y Numérico (NR).

Significado: Resolución de problemas nuevos sin usar palabras. Es el núcleo de la inteligencia informática y de ingeniería.

Reglas de Inferencia:

Si AR es Superior (>80%): El usuario entiende la lógica de sistemas, programación y patrones complejos ("Pensamiento Algorítmico").

Si AR es Bajo: El usuario es concreto y práctico. Necesita instrucciones paso a paso. No poner en roles de desarrollo de software o estrategia.

3.3. El Factor "Eficiencia Administrativa" (PSA + ORT)
Lógica: Cruce de Rapidez (PSA) y Ortografía.

Significado: Capacidad para tareas de oficina, entrada de datos, secretariado y control de calidad.

Reglas de Inferencia:

PSA Alto + Ortografía Alta: El candidato ideal para "Data Entry" o asistencia administrativa. Rápido y sin errores.

PSA Alto + Ortografía Baja: "El Atropellado". Hace mucho trabajo en poco tiempo, pero todo tiene errores. Peligroso en contabilidad o documentación legal.

PSA Bajo + Ortografía Alta: "El Perfeccionista Lento". Calidad excelente, pero baja productividad.

3.4. Detección de Inconsistencias (Lógica de Validez)
El analista debe activar una "Alerta" si detecta estos patrones:

Patrón de Fatiga: Si en la Parte 1 de Rapidez hizo 50 puntos y en la Parte 2 hizo 10. (Se cansó o se aburrió).

Patrón de Azar: Si en Numérico tiene un puntaje estadísticamente igual al azar (ej. 8 puntos sobre 40, marcando 'A' en todas).

Discrepancia Cognitiva: Verbal en Percentil 99 y Abstracto en Percentil 5. (Puede indicar una dificultad específica de aprendizaje o problemas de visión, o que no entendió las instrucciones del Abstracto).

4.4. Recomendación de Roles (Output Final)
Basado en la lógica de la Parte 3, el sistema debe sugerir 3 áreas de desempeño:

Lógica: Si (Numérico > 70%) Y (Abstracto > 70%) -> Sugerir: "Roles Técnicos, Ingeniería, Programación, Análisis Financiero".

Lógica: Si (Verbal > 70%) Y (Rapidez > 60%) -> Sugerir: "Ventas, Atención al Cliente, Recursos Humanos, Comunicación".

Lógica: Si (Rapidez > 80%) Y (Ortografía > 80%) -> Sugerir: "Administración, Operaciones, Logística, Secretariado Ejecutivo".
